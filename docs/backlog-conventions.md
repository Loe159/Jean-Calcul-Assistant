# Conventions du backlog

- Les titres commencent par la phase et la priorité : `[Phase 0][P0]`.
- Les dépendances sont listées avec leurs numéros d’issue GitHub.
- Une issue n’est terminée que lorsque tous ses critères d’acceptation sont validés.
- Priorités : P0 bloquant, P1 important, P2 amélioration.
- Les epics servent de sommaires ; le travail est réalisé dans les issues enfants.
